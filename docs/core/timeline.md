# 全局时间线

全局时间线把**所有** Wiki 词条里的时间线事件聚到一处,按时间排成一张全景图——用来看"这段时间整体发生了什么、各条线索怎么并行推进"。它是一个独立的标签页,点左侧栏的时间线图标打开。

![全局时间线·分支时间树：多条支线按时间轴并行展开](/screenshots/timeline-global-tree.jpg)

::: tip 和词条页里的时间线不一样
[Wiki 词条页](/core/knowledge-engine#wiki-词条页的结构)底部也有一个 Timeline,那只是**这一个词条**相关的事件。这里的**全局时间线**是把每个词条的事件**跨页面汇总**到一张图上看全局。
:::

## 两种视图

右上角切换两种看法,数据相同、呈现不同:

| 视图 | 形态 | 适合 |
|---|---|---|
| **分支时间树** | 横向时间轴,每个词条一条支线,事件点排在轴上 | 看多条线索的并行推进、先后顺序 |
| **放射心智图** | 以中心向外的同心圆,圆环代表时间,事件分布在环上 | 看时间的"远近"分布、整体节奏 |

![全局时间线·放射心智图：同心圆时间环 + 事件分布](/screenshots/timeline-global-radial.jpg)

## 左侧筛选轴

左栏的三组筛选同时作用于两种视图:

| 筛选 | 作用 |
|---|---|
| **语义类型** | 按 Wiki 类型(project / concept / entity …)点击切换显示/隐藏整条支线 |
| **显示分支** | 只看最近活跃的前 N 条支线(按最新事件日期排),避免支线太多 |
| **时间范围** | 拖动滑块限定时间窗口,区间外的事件隐藏,主干保留;默认展示最近三个月 |

## 日期依据:事件日期 vs 记录日期

每个事件有两个时间维度,可二选一定位:

- **事件日期**——这件事**实际发生**的时间(从笔记里抽取)。看"事情本身的时间线"。
- **记录日期**——这件事被**写进知识库**的时间。看"你是什么时候知道/记下的"。

两者切换会重新排布整张图,但不增删事件。

## 时间刻度

`日 / 周 / 月 / 季度` 四档**只改坐标轴的刻度密度**,让你在"看细节"和"看全局"之间缩放,不影响事件本身的数量和位置。
