# Agent Fleet

Agent Fleet 是知我的团队 / 智能体工作区。v0.5 起，你已经可以在应用里看到这个模式；它目前是**本地预览外壳**，用于提前固定入口、布局和未来协作心智。

::: tip 当前状态
v0.5.x 已上线 Fleet 入口、登录门控和 4 个页面骨架；真实云端多 agent 编排、后台任务、团队共享知识空间仍是后续 Pro / Team 能力。
:::

## 是什么

MCP 让 Claude Code / Cursor 读取知我；Agent Fleet 则是反过来，让知我成为指挥位：你在一个入口下发目标，未来由它协调多个 AI agent、文档检索和 MCP 工具，结果再沉淀回你的知识库。

## 进入 Fleet

左侧 ModeRail 里切换到团队 / Agent Fleet 模式。

- 未登录时，会先看到登录门禁。
- 登录后，进入 Fleet 工作区。
- 知识模式仍然不强制登录，本地编辑 / 搜索 / MCP / BYOK AI 都照常可用。

账号说明见 [账号与登录](/start/account)。

## 当前页面

登录后可以看到 4 个 tab：

| Tab | 当前用途 |
|---|---|
| **配置** | 查看运行时、模型、记忆治理等 Fleet 配置卡片。 |
| **消息** | 预览未来多 agent 消息流和知我记忆卡片。 |
| **通讯录** | 预览 agent / 联系人 roster 与详情面板。 |
| **工作台** | 预览任务看板、舰队图、动态和定时任务。 |

这些页面里的数据目前主要是本地演示数据，帮助你理解未来工作流，不代表已经有云端调度或团队协作后台。

## 它和 MCP 的区别

| | [MCP](/integrate/mcp) | Agent Fleet |
|---|---|---|
| 方向 | 外部 agent 读写知我 | 知我协调多个 agent |
| 角色 | 知我是记忆 / 知识层 | 知我是指挥位 |
| 当前状态 | 已可用：默认只读，授权后可提交写入建议 | v0.5.x 预览：入口 + 登录门控 + 工作区外壳 |
| 后续方向 | 更多客户端和工具兼容 | 云端编排、后台任务、团队空间 |

## 未来会做什么

后续 Agent Fleet 会和 Pro / Team 能力结合：

- 云端 Agent Fleet 编排
- 后台任务和定时 Dream-cycle
- 团队共享知识空间
- 成员、权限、角色和审计
- 远程 API / Webhook / 多客户端 Token 管理

这些能力还未在 v0.5.x 上线。当前版本的 Fleet 主要用于提前体验产品方向和工作区结构。

## 今天可以怎么用

在真实 Fleet 编排上线前，推荐这样搭配现有能力：

1. 用 [MCP 接入](/integrate/mcp) 让 Claude Code / Cursor 读写你的知识库。
2. 用 [AI 协作](/ai/collaboration) 在笔记和 Wiki 里就地问答、改进。
3. 用 [Chat 对话](/ai/chat) 进行多轮检索和讨论。

Fleet 会逐步把这些能力收束到一个更高层的指挥入口里。
