# 指挥室

指挥室是 Agent Fleet 的**实时盯盘面** —— 把多个会话并排钉在一起，一眼盯住多个 agent 同时跑，再配一个全局「需要输入」聚合入口。

![指挥室：多窗并排的会话面板 + 顶部「N 个 session 等待输入」HUD](/screenshots/fleet-command-center.jpg)

## 多窗并排

- 从会话列表把任意会话「钉」进指挥室，成为一个并排窗格。
- **横排 / 竖排**一键切换；窗格间有**可拖拽分隔条**调尺寸。
- 窗数无固定上限；空间不足时按**先进先出**挤掉最早钉入的窗。
- 每窗右上角「×」移除；末尾「+」占位新开空窗。
- 排向、尺寸比例、钉住的会话**持久化**，重启恢复。

## 全局「需要输入」HUD

散落在各会话里的权限等待（agent 想执行危险操作、等你批准）会聚合到指挥室顶部：

- 一个「**N 个 session 等待输入**」pill + 可点进对应会话的分组列表。
- 让你不必逐个点开会话，就能一眼扫到"谁在等我"。

## 铁律：钉住 ≠ 派发

::: warning pin ≠ dispatch
把会话钉进指挥室**绝不触发任何派发** —— 不发 prompt、不 resume、不起进程。钉住只是"显示哪些已存在的会话"。派发只能由某个窗格里的输入框显式发送触发。
:::
