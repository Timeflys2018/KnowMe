# 任务看板

任务看板把显式任务组织成看板列 —— 派发、盯进度、看用量、编回知识库。支持看板 / 列表 / 统计三视图。

![任务看板：待派发 / 进行中 / 待审核 / 已完成 四列，任务卡可拖动](/screenshots/fleet-board-kanban.jpg)

## 四列流转

| 列 | 含义 |
|---|---|
| **待派发** | draft（仅记录）+ scheduled（定时未触发）任务；draft 有「立即派发」手动触发 |
| **进行中** | 已派发、runner 正在跑的任务 |
| **待审核** | 需人工确认的产出（审批门控见下方「规划中」） |
| **已完成** | 终态任务（成功 / 失败 / 中止） |

## 派发时机三选

新建任务时选「何时派发」：

- **立即派发**：建任务即刻交给 agent 跑。
- **定时**：选日期 + HH:MM，到点由[调度器](/fleet/autopilots)自动派发（支持跨天）。
- **仅记录（draft）**：只进「待派发」列，不起 runner；之后手动「立即派发」。

## 任务抽屉：转录 + 用量面板

点任意任务卡打开抽屉：

- **转录**：该任务会话的完整对话（与消息 tab 同源）。
- **用量面板**：输入 / 输出 token 数 + 本次 $ 花费。
- **续聊**：抽屉内直接补一句继续该会话。
- **打开产出物**：若任务已沉淀成 wiki 词条，一键跳到 vault 里的产出。

![任务抽屉：转录 + 用量面板（token / 花费）+ 续聊输入框](/screenshots/fleet-board-drawer.jpg)

## 看板 ↔ 消息双向跳转

- 任务卡 → 抽屉「查看会话」→ 跳[消息](/fleet/messages) tab 恢复该会话。
- 消息 tab 会话行的「任务」徽标 → 跳回看板并打开对应任务抽屉。

## 降为普通会话（撤销 task 状态）

误在闲聊里 `/task`、或事后觉得某任务只是随手一问？任务卡右键「**降为普通会话**」：

- 摘掉 task 标记 → 退出看板、会话列表「任务」徽标消失。
- **完整保留该会话及全部对话历史**，之后当普通对话继续用。

::: tip 降级 ≠ 删除
「降为普通会话」只删 task 记录、保留会话与历史；「删除任务」才级联删除整个会话（不可撤销）。二次确认弹窗文案会明确区分。
:::
