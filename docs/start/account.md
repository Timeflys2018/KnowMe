# 账号与登录

知我 v0.5 起加入账号系统。账号用于进入 Agent Fleet（团队模式）和未来的 Pro / 云同步 / 协作能力；**本地核心功能不强制登录**。

## 需要账号吗？

不需要。以下能力无需登录即可使用：

- 本地编辑、搜索、Wiki 编译、知识图谱、全局时间线
- 本地文件、外部挂载、导出
- BYOK / 本地 LLM
- 本地 MCP server

需要账号的场景：

- 进入 **Agent Fleet / 团队模式**
- 未来开启云同步、分享、团队协作、Pro 权益等云端能力
- 管理未来和账号绑定的设备、订阅、权益

## 注册账号

当前支持 **邮箱 + 密码** 注册。

1. 点击左侧 ModeRail 底部的账号入口，或进入 Agent Fleet 时按提示登录。
2. 选择注册，填写邮箱和密码。
3. 密码至少 12 位；知我不强制大小写、数字、符号组合。
4. 查收邮箱验证码，输入验证码完成邮箱验证。

::: tip 收不到验证码？
先检查垃圾邮件；确认邮箱拼写无误后再重发。重发有冷却时间，短时间连续请求会被限流。
:::

## 登录与免登录

登录后，知我会把 refresh token 存在系统安全存储中（macOS Keychain / Windows Credential Vault）。下次打开应用时会自动续期，无需重新输入密码。

如果安全存储不可用，知我仍可使用，但关闭应用后可能需要重新登录。

## 退出登录

账号菜单里选择退出登录。退出会同时清除本地登录态，并让服务端作废当前设备的会话。

## 找回密码

忘记密码时，使用登录面板里的找回入口：

1. 输入注册邮箱。
2. 查收密码找回验证码。
3. 设置新密码。

为防止邮箱枚举，知我不会明确告诉你“邮箱是否存在”。如果邮箱已注册，会收到验证码；如果没有收到，请检查拼写、垃圾箱和重发冷却。

## 设备与安全

每次成功登录都会登记为一个设备。未来设备管理会用于查看和吊销登录设备。

安全边界：

- 明文密码不会发送到服务器。
- 密码派生在本机完成，服务器只保存验证所需的派生结果。
- 账号 API 使用 `https://api.useknowme.com`。
- 账号不改变本地优先原则：你的笔记内容不会因为登录而上传。

## 常见问题

### v0.5.0 注册/登录提示网络失败怎么办？

请升级到 v0.5.1 或更高版本。v0.5.1 已把账号服务切到稳定域名 `api.useknowme.com`。

### 不登录会影响本地笔记吗？

不会。本地编辑、搜索、Wiki、MCP、BYOK AI 都照常可用。

### 登录后笔记会上传吗？

不会。账号只用于身份、未来权益和云端能力入口。当前本地笔记仍保存在你的电脑上。
